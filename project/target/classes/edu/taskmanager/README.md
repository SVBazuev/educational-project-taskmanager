# Дорожная карта (этапы разработки MVP)
Этапы построены так, чтобы постепенно внедрять паттерны, начиная с простого и наращивая сложность. Каждый этап должен завершаться работоспособным кодом, который можно протестировать.

## Этап 0: Настройка проекта
- Создать структуру пакетов согласно описанию.

- Настроить систему сборки (Maven/Gradle) — можно не обязательно, просто как проект в IDE.

- Создать перечисления `Priority` и `TaskStatus`.

## Этап 1: Модели и репозиторий (без паттернов)
- Реализовать классы `Task`, `Project`, `User`, `Tag` с полями, конструкторами, геттерами/сеттерами.

- Реализовать интерфейс `TaskRepository` и класс `InMemoryTaskRepository` с методами `save`, `findById`, `findAll`, `deleteById`. Использовать `ConcurrentHashMap` и `AtomicLong`.

- Написать простой тест в `main` для создания задачи и сохранения в репозиторий.

## Этап 2: Builder для Task
- Реализовать `TaskBuilder` с методами для всех полей.

- Добавить в `TaskBuilder` проверки обязательных полей и создание объекта.

- Продемонстрировать создание задачи через билдер в `main`.

## Этап 3: Стратегия сортировки
- Создать интерфейс `TaskSortingStrategy` и несколько реализаций (по сроку, по приоритету).

- Реализовать `TaskManagerService`, который принимает `TaskRepository` и использует стратегию.

- Добавить метод `getAllTasksSorted` в сервис.

- В `main` показать переключение стратегий.

## Этап 4: Цепочка обязанностей для фильтрации
- Реализовать интерфейс `TaskFilter` и конкретные фильтры (`StatusFilter`, `TagFilter`, `ProjectFilter`).

- Реализовать `FilterChain` для связывания фильтров и применения к списку задач.

- В `TaskManagerService` добавить поле `filterChain` и метод `getFilteredTasks`.

- Протестировать комбинации фильтров.

## Этап 5: Прокси (защитный)
- Создать интерфейс `TaskService` с методами CRUD, принимающими `User`.

- Реализовать `TaskServiceImpl`, который использует `TaskRepository`.

- Реализовать `TaskServiceProxy`, который добавляет проверку прав доступа (простую логику: например, пользователь с ролью ADMIN может всё, остальные только свои задачи).

- Модифицировать `TaskManagerService` так, чтобы он принимал `TaskService` (интерфейс) и через него работал.

- В `main` показать работу с прокси.

## Этап 6: Декоратор для задачи
- Создать абстрактный класс `TaskDecorator`, делегирующий методы обёрнутой задаче.

- Реализовать `LoggingTaskDecorator`, который логирует изменения.

- Показать, как можно обернуть существующую задачу декоратором и вызвать сеттеры, увидеть логи.

## Этап 7: Адаптер для экспорта
- Создать интерфейс `TaskExporter` и реализации `CsvTaskExporter`, `JsonTaskExporter`.

- Добавить в `TaskManagerService` метод `exportTasks(TaskExporter exporter)`.

- Продемонстрировать экспорт задач в CSV и JSON.

## Этап 8: Адаптер для календаря (имитация)
- Создать интерфейс `ExternalCalendarAdapter` и простую имитацию `GoogleCalendarAdapter` (вывод в консоль).

- Показать экспорт задачи в календарь.

## Этап 9: Интеграция всех частей в консольное приложение
- В `main` создать несколько задач через билдер.

- Показать:
  
  - Сортировку разными стратегиями.

  - Фильтрацию через цепочку.

  - Использование защитного прокси с разными пользователями.

  - Логирование через декоратор.

  - Экспорт в CSV/JSON.

  - Синхронизацию с календарём.

## Этап 10 (дополнительно): Виртуальный прокси для подзадач
- Модифицировать модель `Task`, чтобы добавить поле `parentId` или связь с родителем.

- Реализовать `LazyLoadingTaskProxy`, который загружает подзадачи только при вызове `getSubtasks()`.

- Продемонстрировать ленивую загрузку.

# Эта дорожная карта позволит вам последовательно освоить каждый паттерн в контексте реального проекта, при этом на каждом шаге проект остаётся рабочим и тестируемым.
